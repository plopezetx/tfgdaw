# 06 - Guion de Demo del TFG

## Objetivo

Mostrar de forma clara el flujo principal del sistema durante la defensa.

## Flujo de demo

### 1. Inicio

- Abrir la aplicación.
- Mostrar landing o pantalla inicial.
- Iniciar sesión.

### 2. Creación de proyecto

- Crear nuevo proyecto.
- Seleccionar plantilla web.
- Abrir el IDE.

### 3. Edición

- Mostrar árbol de archivos.
- Abrir archivo principal.
- Modificar código.

### 4. Ejecución

- Pulsar Run.
- Mostrar terminal/logs.
- Mostrar preview.

### 5. Guardado

- Guardar proyecto.
- Cerrar sesión o recargar.
- Volver a abrir proyecto.
- Comprobar que los cambios siguen ahí.

### 6. Publicación

- Marcar proyecto como público.
- Abrir página pública.
- Mostrar código o preview pública.

### 7. Fork/remix

- Crear una copia del proyecto.
- Editar la copia.
- Comprobar que es independiente.

### 8. Asistente IA

- Seleccionar un bloque de código.
- Pedir explicación.
- Generar un snippet o sugerencia.
- Mostrar que la respuesta se integra en el flujo del IDE.

## Cierre

Explicar brevemente:

- Por qué se ha elegido ejecución local en navegador.
- Qué limitaciones tiene el sistema.
- Qué mejoras futuras se plantean.
